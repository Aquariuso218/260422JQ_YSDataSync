## 一、中间表设计

### 1.收付款单中间表 (`EF_MidYSBillData`)

由于表体会有多行，推荐将**表体ID (`id`)** 设置为主键，表头ID (`headid`) 作为关联外键。

#### 表结构设计

以此表格字段描述为准SQL Server 建表脚本

```sql
CREATE TABLE [dbo].[EF_MidYSBillData] (
    [id] VARCHAR(50) NOT NULL PRIMARY KEY,           -- 主键，子表ID
    [headid] VARCHAR(50) NULL,                       -- 表头ID
    [code] VARCHAR(50) NULL,                         -- 单据编码
    [billDate] DATE NULL,                            -- 单据日期
    [creatorUserName] VARCHAR(50) NULL,              -- 制单人
    [orgCode] VARCHAR(50) NULL,                      -- 组织编码
    [depcode] VARCHAR(50) NULL,                      -- 部门编码
    [staffCode] VARCHAR(50) NULL,                    -- 业务员编码
    [enterpriseBankAccountNo] VARCHAR(100) NULL,     -- 企业单位银行账号 
    [enterpriseBankAccountName] VARCHAR(100) NULL,   -- 企业单位银行名称
    [settleModeCode] VARCHAR(50) NULL,               -- 结算方式编码
    [settleState] INT NULL,                          -- 结算状态 (0:不结算; 1:待结算; 3:全部成功; 4:部分成功; 5:全部止付)
    [quickTypeName] VARCHAR(100) NULL,               -- 款项类型名称
    [cVouchType] VARCHAR(50) NULL,                   -- 单据类型 (根据接口判断是付款单、付款退款单、收款单、收款退款单)
    [objectCode] VARCHAR(50) NULL,                   -- 对象编码 (付款单取supplierCode；收款单取customerCode)
    [objectName] VARCHAR(200) NULL,                  -- 对象名称 (付款单取supplierName；收款单取customerName)
    [objectBankAccountName] VARCHAR(100) NULL,       -- 对象账户名称
    [objectBankAccountNo] VARCHAR(100) NULL,         -- 对象银行账号
    [LocalTaxIncludedAmount] DECIMAL(18,4) NULL,     -- 金额
    
    -- 对接控制字段
    [createTime] DATETIME DEFAULT GETDATE(),         -- 数据拉取写入中间表的时间
    [UpdateTime] DATETIME NULL,         			 -- 更新已有数据时间
    [processStatus] INT DEFAULT 0,                   -- 0:未处理, 1:写入U8成功, 2:写入U8失败
    [processMsg] NVARCHAR(500) NULL,                 -- 写入U8失败时的报错信息
    [u8code] NVARCHAR(100) NULL,                 	 -- 写入U8成功后的单据号
    [SYNTime] DATETIME NULL					 		 -- 写入U8成功时的时间
);

-- 建立索引：用于快速查询未处理的数据
CREATE INDEX [IX_EF_MidYSBillData_status] ON [dbo].[EF_MidYSBillData]([processStatus]);
```

### 2. 数据同步日志表 (`EF_sysSyncLog`)

这张表用于记录外部程序每次调用接口抓取数据的情况，主要用于记录 `open_billDate_begin` 和 `open_billDate_end`，作为增量抓取的依据，并监控接口健康状态。

#### 表结构设计

| **字段名**        | **数据类型** | **备注**                                               |
| ----------------- | ------------ | ------------------------------------------------------ |
| `log_id`          | INT          | 主键，自增                                             |
| `interface_name`  | VARCHAR      | 接口标识 (例如：'YS_Payment')                          |
| `sync_start_time` | DATETIME     | 本次抓取的业务时间范围（起）对应 `open_billDate_begin` |
| `sync_end_time`   | DATETIME     | 本次抓取的业务时间范围（止）对应 `open_billDate_end`   |
| `total_records`   | INT          | 本次抓取到的总条数（对应返回体中的 `recordCount`）     |
| `sync_status`     | INT          | 接口调用状态 (1:成功, 0:异常)                          |
| `error_msg`       | NVARCHAR     | 如果接口调用异常，记录具体报错信息                     |
| `execute_time`    | DATETIME     | 记录本次任务的实际执行时间                             |

#### SQL Server 建表脚本

```
CREATE TABLE [dbo].[EF_sysSyncLog] (
    [log_id] INT IDENTITY(1,1) NOT NULL PRIMARY KEY,
    [interface_name] VARCHAR(50) NOT NULL,
    [sync_start_time] DATETIME NOT NULL,
    [sync_end_time] DATETIME NOT NULL,
    [total_records] INT DEFAULT 0,
    [sync_status] INT DEFAULT 1, -- 1:成功, 0:失败
    [error_msg] NVARCHAR(MAX) NULL,
    [execute_time] DATETIME DEFAULT GETDATE()
);

-- 建立索引以便快速查询最后一次成功的抓取时间
CREATE INDEX [IX_EF_sysSyncLog_name_time] ON [dbo].[EF_sysSyncLog]([interface_name], [execute_time] DESC);
```

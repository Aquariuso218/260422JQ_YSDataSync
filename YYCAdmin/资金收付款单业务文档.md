

# 资金收付款单业务文档

## AI提示词描述（供代码生成工具参考）：

【角色设定】
你是一个资深的 .NET 后端开发工程师，熟悉 C#、.NET Core、SQL Server 以及 ZR.Admin (基于 SqlSugar 或 EF Core) 架构的开发。具备丰富的三方 API 对接和高并发数据同步经验。

【核心任务】
	对于已同步到中间表的数据需要一个刷新的定时任务，用于查询中间表中settlestatus不等于2，即未结算完成的数据，查询其是否在YS系统中状态是否已经变为已结算（1），实现刷新数据。

​	同时调整已完成程序中，查询数据时如果该数据已经在中间表中，不需要现有的update操作，直接跳过该条数据的处理即可

**【备注：后续 U8 写入程序的过滤规则，不在本次抓取任务范围内】** 

本次抓取任务只负责将已审核的单据拉取到中间表 `EF_MidYSBillData`。 后续由中间表写入 U8 的程序需遵循以下规则：取已结算完成 (`settlestatus = 2`) 的数据，进行写入操作。

## 一、中间表设计

### 1.收付款单中间表 (`EF_MidYSBillData`)

由于表体会有多行，推荐将**表体ID (`id`)** 设置为主键，表头ID (`headid`) 作为关联外键。

#### 表结构设计

以此表格字段描述为准

| **字段名**                  | **数据类型** | **长度** | **来源**（表体为FundPayment_b）                              | **备注**                                 |

| `id`                        |   VARCHAR    | 50       | 表体 `id`                                                    | **主键**，子表ID                         |
| `mainid`                    |   VARCHAR    | 50       | 表体 `mainid`                                                | 表头ID                                   |
| `code`                      |   VARCHAR    | 50       | 表头 `code`                                                  | 单据编码                                 |

#### SQL Server 建表脚本

```sql
CREATE TABLE [dbo].[EF_MidYSBillData] (
    [id] VARCHAR(50) NOT NULL PRIMARY KEY,           -- 主键，子表ID (表体id)
    [mainid] VARCHAR(50) NULL,                       -- 表头ID (表体mainid)
    [code] VARCHAR(50) NULL,                         -- 单据编码
    [billDate] DATE NULL,                            -- 单据日期
    [creator] VARCHAR(50) NULL,                      -- 制单人名
    [orgCode] VARCHAR(50) NULL,                      -- 组织编码 (需通过accentity ID查询)
    [depcode] VARCHAR(50) NULL,                      -- 部门编码 (需通过dept ID查询)
    [operatorCode] VARCHAR(50) NULL,                 -- 业务员编码 (需通过operator ID查询)
    [enterpriseBankAccountNo] VARCHAR(100) NULL,     -- 企业单位银行账号 (需通过ID查询)
    [enterpriseBankAccountName] VARCHAR(100) NULL,   -- 企业单位银行账户名称 (需通过ID查询)
    [settleModeCode] VARCHAR(50) NULL,               -- 结算方式编码 (需通过settlemode ID查询)
    [settlestatus] INT NULL,                         -- 结算状态 (1:未结算, 2:已结算)
    [quickTypeName] VARCHAR(100) NULL,               -- 款项类型名称
    [cVouchType] VARCHAR(50) NULL,                   -- 单据类型 (付款单/收款单等)
    [caobject] INT NULL,							 -- 对象类型 对象类型(1：客户，2：供应商）
    [objectCode] VARCHAR(50) NULL,                   -- 对象编码 (需通过oppositeobjectid ID查询供应商/客户编码)
    [oriSum] DECIMAL(18,4) NULL,                     -- 金额
    
    -- 对接控制字段
    [createTime] DATETIME DEFAULT GETDATE(),         -- 数据拉取写入中间表的时间
    [updateTime] DATETIME NULL,                      -- 更新已有数据时间
    [processStatus] INT DEFAULT 0,                   -- 0:未处理, 1:写入U8成功, 2:写入U8失败
    [processMsg] NVARCHAR(500) NULL,                 -- 写入U8失败时的报错信息
    [u8code] VARCHAR(100) NULL,                      -- 写入U8成功后的单据号
    [SYNTime] DATETIME NULL                          -- 写入U8成功时的时间
);

-- 索引以优化U8写入程序抓取效率
CREATE INDEX [IX_EF_MidYSBillData_status] ON [dbo].[EF_MidYSBillData]([processStatus]);
```

### 2. 数据同步日志表 (`EF_sysSyncLog`)

这张表用于记录外部程序每次调用接口抓取数据的情况，主要用于记录 `open_billDate_begin` 和 `open_billDate_end`，作为增量抓取的依据，并监控接口健康状态。

#### 表结构设计

| **字段名**        | **数据类型** | **备注**                                               |
| ----------------- | ------------ | ------------------------------------------------------ |
| `log_id`          | INT          | 主键，自增                                             |
| `interface_name`  | VARCHAR      | 接口标识 (例如：'YS_Payment')                          |
| `sync_start_time` | DATETIME     | 本次抓取的业务时间范围（起）对应 `open_billDate_begin` |
| `sync_end_time`   | DATETIME     | 本次抓取的业务时间范围（止）对应 `open_billDate_end`   |
| `total_records`   | INT          | 本次抓取到的总条数（对应返回体中的 `recordCount`）     |
| `sync_status`     | INT          | 接口调用状态 (1:成功, 0:异常)                          |
| `error_msg`       | NVARCHAR     | 如果接口调用异常，记录具体报错信息                     |
| `execute_time`    | DATETIME     | 记录本次任务的实际执行时间                             |

#### SQL Server 建表脚本

```
CREATE TABLE [dbo].[EF_sysSyncLog] (
    [log_id] INT IDENTITY(1,1) NOT NULL PRIMARY KEY,
    [interface_name] VARCHAR(50) NOT NULL,
    [sync_start_time] DATETIME NOT NULL,
    [sync_end_time] DATETIME NOT NULL,
    [total_records] INT DEFAULT 0,
    [sync_status] INT DEFAULT 1, -- 1:成功, 0:失败
    [error_msg] NVARCHAR(MAX) NULL,
    [execute_time] DATETIME DEFAULT GETDATE()
);

-- 建立索引以便快速查询最后一次成功的抓取时间
CREATE INDEX [IX_EF_sysSyncLog_name_time] ON [dbo].[EF_sysSyncLog]([interface_name], [execute_time] DESC);
```

#### （3）数据对照

直接在详情接口中能够查询到基础数据。针对带有 ID 的字段，需要**在程序中调用相应的 YS 基础档案接口，将 ID 转换为对应的 Code 或名称**，然后与表头、表体其它数据组合并处理逻辑后，写入中间表 `EF_MidYSBillData`。

| **中间表字段名**              | **数据类型** | **YS接口来源字段 (表体为 FundPayment_b)** | **转换与处理逻辑 / 备注**                                    |
| ----------------------------- | ------------ | ----------------------------------------- | ------------------------------------------------------------ |
| **id**                        | VARCHAR      | 表体 `id`                                 | **主键**，子表ID直接取值                                     |
| **mainid**                    | VARCHAR      | 表体 `mainid`                             | 表头ID，直接取值                                             |
| **code**                      | VARCHAR      | 表头 `code`                               | 单据编码，直接取值                                           |
| **billDate**                  | DATE         | 表头 `billDate`                           | 单据日期，直接取值                                           |

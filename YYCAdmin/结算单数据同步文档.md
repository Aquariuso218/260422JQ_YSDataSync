# 数据同步文档

## AI提示词描述：

【核心任务】

​	实现贴现办理、到期兑付、银行托收的数据抓取，一样从ys抓取到EF_MidYSBillData

【具体执行逻辑】

1. 增量时间计算：

   * 每次执行任务前，先查询数据同步日志表 `EF_sysSyncLog` 中该接口最后一次执行成功的 `sync_end_time`。
   * 将该时间作为本次列表查询的 `open_vouchdate_begin`，当前时间作为 `open_vouchdate_end`。
2. 次子任务只需调一次接口，无需列表、详情去查
3. 主要参照三个业务的对照表，从返回体中取出数据到中间表
4. 日志记录兜底：
   * 必须包含完整的 try-catch 处理机制。任务结束时向 `EF_sysSyncLog` 写入一条日志（记录本次抓取的起止时间、总条数、成功/失败状态 `processStatus` 及错误信息 `processMsg`）。

【优化任务】：

​	1.结算单数据刷新程序优化，如果刷新时调用接口查询查询数据为空，或者子表id未匹配到，则直接删除中间表该结算单行







## 一、中间表设计

### 1.中间表 (`EF_MidYSBillData`)

由于表体会有多行，推荐将**表体ID (`id`)** 设置为主键，表头ID (`mainid`) 作为关联外键。

### 2.sql

```sql
CREATE TABLE [dbo].[EF_MidYSBillData] (
    [autoId] INT IDENTITY(1,1) NOT NULL PRIMARY KEY, -- 自增主键，autoId
    [ID] VARCHAR(50) NULL,            					  -- 主键，子表ID
    [mainid] VARCHAR(50) NULL,                        -- 表头ID
    [cVouchCode] VARCHAR(50) NULL,                    -- 单据编码
    [billDate] DATE NULL,                             -- 单据日期
    [CMAKER] VARCHAR(50) NULL,                        -- 制单人名
    [orgCode] VARCHAR(50) NULL,                       -- 组织编码
    [cDepCode] VARCHAR(50) NULL,                      -- 部门编码
    [cNatBankAccount] VARCHAR(100) NULL,              -- 企业单位银行账号
    [cNatBank] VARCHAR(100) NULL,                     -- 企业单位银行账户名称
    [cSSName] VARCHAR(50) NULL,                       -- 结算方式名称
    [settlestatus] INT NULL,                          -- 结算状态 (1:未结算、3:已结算)
    [quickTypeName] VARCHAR(100) NULL,                -- 款项类型名称
    [cVouchType] VARCHAR(50) NULL,                    -- 单据类型 (转换：1-资金收款；2-资金付款)
    [cDwType] VARCHAR(100) NULL,                      -- 对象类型 (转换：1-客户，2-供应商)
    [cDwCode] VARCHAR(100) NULL,                      -- 对象编码
    [iAmount] DECIMAL(18,4) NULL,                     -- 金额
    [cNoteCode] VARCHAR(100) NULL,                    -- 票据号
    [tradetypeName] VARCHAR(100) NULL,                -- 来源交易类型
    discountInterest VARCHAR(100) NULL,     		  -- 利息
    noteTypeCode VARCHAR(100) NULL,    				  -- 票据类型
    
    
    -- 对接控制字段
    [createTime] DATETIME DEFAULT GETDATE(),          -- 数据拉取写入中间表的时间
    [updateTime] DATETIME NULL,                       -- 数据状态刷新时间
    [processStatus] INT DEFAULT 0,                    -- 0:未处理, 1:写入U8成功, 2:写入U8失败
    [processMsg] NVARCHAR(500) NULL,                  -- 写入U8失败时的报错信息
    [u8code] VARCHAR(100) NULL,                       -- 写入U8成功后的单据号
    [SYNTime] DATETIME NULL                           -- 写入U8成功时的时间
);

-- 创建索引：优化U8写入程序抓取“未处理”数据时的查询性能
CREATE INDEX [IX_EF_MidYSBillData_status] ON [dbo].[EF_MidYSBillData]([processStatus]);
```
